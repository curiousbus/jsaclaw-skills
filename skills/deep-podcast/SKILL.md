# Deep Podcast Skill

将**长文章或深度话题**转换为 5-10 分钟的双人深度讨论播客。

## 与普通播客的区别

| 特性 | 普通新闻播客 | 深度话题播客 |
|------|-------------|-------------|
| 内容 | 多条新闻快速浏览 | 单个话题深入讨论 |
| 时长 | 3-5 分钟 | 5-10 分钟 |
| 对话段数 | 30-50 段 | 80-120 段 |
| 讨论深度 | 简要说明 | 3-5 轮反复追问 |
| 开场 | 问候+自我介绍 | 直接切入话题 |
| 结尾 | 简单再见 | 升华+令人深思的问题 |

## 触发条件

用户以下任一情况时使用本 skill：

- 明确要求「深度讨论」「深入聊聊」某个话题
- 提供长文章（>500 字）要求做成播客
- 说「做成深度播客」「生成深度对话」
- 要求对单个话题进行多角度分析

## 播客风格

### 主持人（飞飞）
- 独立解读能力强，有自己主见
- 会读空气，知道什么时候追问、什么时候调侃
- 风格犀利但不刻薄，敢挑战权威观点
- 对技术有自己的判断力，不轻易被带节奏

### 专家（卡帕东）
- 名字致敬 Karpathy，但更接地气、更幽默
- 非常擅长用类比把复杂技术讲得通俗易懂
- 直接、坦诚，不装高手
- 会讲很多自己的真实感受和经历
- 展现脆弱一面：对 AI 时代快速变化的焦虑、对技术浪潮的无力感
- 像身边的一个大哥/带头大哥，让人感到亲切、不孤独
- 面对变化时既有焦虑也有乐观，很真实

### 对话风格
- 两人之间有化学反应，飞飞负责追问和挑战
- 卡帕东负责深度解读和类比
- 对话要自然，像两个老朋友在聊技术，不是在念稿

### 对话结构
1. **开场**：直接抛出引人注目的观点，无问候无自我介绍
2. **核心论点讨论**：3-4 个论点，每个展开 3-5 轮深度讨论
3. **综合阶段**：把不同观点放在一起，寻找矛盾和联系
4. **挑战追问**：主持人质疑专家，专家反问主持人
5. **深入挖掘**：重新审视前提假设
6. **实践层面**：理论如何落地
7. **展望未来**：趋势判断
8. **结尾升华**：提出令人深思的问题

## 使用方式

### 命令行

```bash
python3 ~/.openclaw/scripts/deep_podcast_tts.py <output_path> "<话题内容>"
```

示例：
```bash
python3 ~/.openclaw/scripts/deep_podcast_tts.py "ai_ethics.mp3" "人工智能的伦理问题是一个非常复杂的议题，涉及到隐私、公平性、透明度等多个维度..."
```

### 通过 Claude Code

直接说：
- 「把这篇文章做成深度播客」
- 「深入聊聊 AI 伦理问题」
- 「生成一个关于远程工作的深度讨论播客」

## 输出

- 音频文件：`~/.openclaw/media/outbound/<name>.mp3`
- **脚本文件**：`~/.openclaw/media/outbound/<name>_script.txt`（对话文字稿）
- 音质：PCM 无损中间格式 + loudnorm 标准化（I=-16:TP=-5.0:LRA=6）+ 最终 128kbps MP3 编码
- TTS 编码：默认使用 PCM 模式（`encoding="pcm"`），避免短片段独立 MP3 编码的帧边界伪影
- 句尾标点自动补全：clean_tts_text() 确保每段文本末尾有标点，防止 TTS 句尾爆音
- 句首笑声自动删除：去除"呵/哈哈/呵呵"等独立句首笑声，防止爆音
- 峰值限制 TP=-5.0：防止特定字词的 TTS 情感峰值导致爆音
- 尾部 50ms 静音缓冲（apad=pad_dur=0.05）：段间过渡更平滑

## 📁 文件命名规范

**深度播客**：`deep_<主题关键词>.mp3`
- 用英文简短关键词，下划线分隔
- 例：`deep_iphone17_400b_llm.mp3`、`deep_chatgpt_checkout_conversion.mp3`
- 脚本文件同前缀：`deep_iphone17_400b_llm_script.txt`
- 关键词从话题中提取 2-4 个最能概括的核心词

**早晚报播客**：`morning_<日期>_<头条关键词>.mp3` / `evening_<日期>_<头条关键词>.mp3`
- 例：`morning_20260324_iphone17_claude_code.mp3`
- 日期格式：YYYYMMDD
- 头条关键词：从当日简报头条新闻提取 1-3 个核心关键词
- 脚本文件同前缀：`morning_20260324_iphone17_script.txt`

**生成时由调用方（agent/cron）指定文件名**，不要使用默认名或时间戳。

## ⚠️ 重要：脚本保存规则

**每次生成深度播客时，必须同时生成并发送对话脚本文件：**

1. 脚本由 `deep_podcast_tts.py` 自动保存（与音频同目录，后缀 `_script.txt`）
2. 发送音频时，**必须同时发送脚本文件**（用 message tool 的 filePath 参数）
3. 脚本格式：每段对话一行 `[N/M] role: 内容`
4. 脚本用途：供用户查看、前端字幕、AI 重点提醒等

**发送模板**：
```
音频文件 → filePath: media/outbound/<name>.mp3
脚本文件 → filePath: media/outbound/<name>_script.txt
```

如果脚本未自动保存（老版本），需手动从日志中提取对话内容并保存。
